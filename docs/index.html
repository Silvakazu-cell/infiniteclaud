Problema vs Solucao

Por que Computer Use nao e suficiente — e como InfiniteClaud resolve.

O Problema: Computer Use

  • Lento: 2-5 segundos por acao
  • Caro: +500 tokens por acao (screenshots)
  • Impreciso: baseado em pixels, falha com resolucoes diferentes
  • Sem acesso remoto: precisa de tela ativa
  • Sem inteligencia de roteamento de modelos

A Solucao: InfiniteClaud

  • DOM direto: acessa elementos sem screenshots
  • APIs nativas: AppleScript, Shell, Playwright
  • MCP Protocol: integracao nativa com Claude Code
  • Zero overhead: sem processamento de imagem
  • Model Router: escolhe o modelo ideal por tarefa

20 Ferramentas Especializadas

Catalogo completo organizado por dominio.

🌐 Web (Playwright) 8 ferramentas
  • web_navigate — navegar para URL
  • web_click — clicar em elemento
  • web_type — digitar em campo
  • web_extract — extrair conteudo da pagina
  • web_screenshot — capturar tela
  • web_scroll — scroll na pagina
  • web_wait — aguardar elemento
  • web_evaluate — executar JavaScript
📁 Filesystem 5 ferramentas
  • fs_read — ler arquivo
  • fs_write — escrever arquivo
  • fs_list — listar diretorio
  • fs_search — buscar arquivos
  • fs_watch — monitorar mudancas

  • native_click — clicar em coordenada
  • native_type — digitar texto
  • native_hotkey — atalho de teclado
  • native_screenshot — captura de tela
  • native_move — mover mouse

FAQ

Perguntas frequentes sobre o InfiniteClaud.

Preciso de API key?▼
Nao. O InfiniteClaud funciona diretamente com sua assinatura do Claude Code. Sem API keys, sem configuracao extra.
Funciona em Windows?▼
Atualmente funciona em macOS. Suporte a Linux esta em desenvolvimento e sera lancado em breve. Windows esta no roadmap futuro.
E gratuito?▼
Sim, 100% gratuito e open source sob licenca MIT. Use, modifique e distribua livremente.
Como atualizo?▼
Execute claude plugin update infiniteclaud no terminal.
